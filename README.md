# მისაღები გამოცდების ანალიტიკის სისტემა

უმაღლეს საგანმანათლებლო პროგრამებზე მისაღები გამოცდების მონაცემთა ანალიტიკური პლატფორმა. სისტემა აერთიანებს ჩარიცხვების, აბიტურიენტთა პრიორიტეტებისა და უნივერსიტეტების ცნობარის მონაცემებს, აანალიზებს მათ და ვიზუალურად წარმოაჩენს პროგრამების „ჯანმრთელობას", მოთხოვნადობას, კონკურენციასა და ბაზრის ტენდენციებს.

## რა შეუძლია სისტემას

- 📥 **მონაცემთა იმპორტი** სამი წყაროდან — ჩარიცხულთა სია (Excel), პრიორიტეტები (Excel) და ცნობარი (PDF);
- 🩺 **ჯანმრთელობის ინდექსი** — თითო პროგრამის კომპოზიტური შეფასება მოთხოვნის, შევსების, პრიორიტეტის ხარისხისა და ფასის მიხედვით;
- 📊 **ანალიტიკა** — ჩარიცხვის პროგნოზი, კონვერსია, ფასისადმი მგრძნობელობა, მრავალწლიანი ტენდენციები;
- 🏛️ **ბაზრის ანალიზი** — დეფიციტური მიმართულებებისა და კონკურენციის გამოვლენა;
- ⚖️ **პროგრამების შედარება** — 5-მდე პროგრამის გვერდ-გვერდ შეპირისპირება.

## ტექნოლოგიური სტეკი

**Backend:** .NET 10 · ASP.NET Core · Entity Framework Core · SQL Server · MediatR (CQRS) · FluentValidation
**Frontend:** React 19 · Vite · TypeScript · Tailwind CSS · TanStack Query · Recharts
**ფაილების დამუშავება:** ClosedXML (Excel) · PdfPig (PDF)

არქიტექტურის, ანალიტიკური ალგორითმებისა და მონაცემთა მოდელის დეტალური აღწერისთვის იხ. [ტექნიკური დოკუმენტაცია](./docs/README.md).

## სწრაფი გაშვება

### წინაპირობები
- .NET SDK 10.0+
- Node.js 20+
- Microsoft SQL Server (ან LocalDB / Express)

### 1. სერვერის (API) გაშვება

```bash
cd src/Admissions.Api
dotnet run
```
API ხელმისაწვდომი იქნება მისამართზე **http://localhost:5116** (Swagger: `/swagger`).
ბაზა იქმნება ავტომატურად პირველ გაშვებაზე — ხელით შექმნა საჭირო არ არის.

> კავშირის სტრიქონი მითითებულია `src/Admissions.Api/appsettings.json`-ში. საჭიროებისამებრ შეცვალეთ SQL Server-ის ინსტანციის სახელი.

### 2. Frontend-ის გაშვება (ცალკე ტერმინალში)

```bash
cd web
npm install
npm run dev
```
აპლიკაცია გაიხსნება მისამართზე **http://localhost:5173**.

### 3. სატესტო მონაცემების გენერაცია (არასავალდებულო)

```bash
dotnet run --project tools/MockGen -- ./mock-data
```
შექმნის სატესტო `.xlsx` ფაილებს 2023–2025 წლებისთვის, რომლებიც შემდეგ შეგიძლიათ ატვირთოთ „მონაცემთა მართვის" გვერდიდან.

## პროექტის სტრუქტურა

```
├── src/
│   ├── Admissions.Domain/          # ბიზნეს-არსებები და enum-ები
│   ├── Admissions.Application/     # CQRS, ანალიტიკა, ვალიდაცია
│   ├── Admissions.Infrastructure/  # EF Core, ფაილების პარსერები, მიგრაციები
│   └── Admissions.Api/             # ASP.NET Core REST API
├── tests/Admissions.Tests/         # უნიტ- და ინტეგრაციული ტესტები
├── web/                            # React + Vite Frontend
├── tools/MockGen/                  # სატესტო Excel ფაილების გენერატორი
└── docs/                           # ტექნიკური დოკუმენტაცია
```
